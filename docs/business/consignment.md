# 寄售与托管

## 三种销售模式对比

| 概念 | 货的归属 | 谁来销售 | 谁赚佣金 |
|------|---------|---------|---------|
| **寄售商** | EBUY 的货 | 寄售商 | 寄售商赚 |
| **托管商** | 托管商的货 | EBUY 系统 | EBUY 赚 |
| **合作商** | 合作商的货 | 合作商自己 | 合作商 |

## 合作商 vs 托管商的区别

**合作商**：只是供货。

**托管商**：把整个业务都交给 EBUY 系统来管：
- 销售
- 配单送货
- 售后

## 寄售订单流程

```
零售管理 → 寄售订单的快捷下单
    ↓
导出模板 → 填入下单数量 → 上传模板 → 完成下单
    ↓
员工端 → 寄售商配单模块 → 配单
```

## 寄售相关表

- **ConsigneeOrder**：寄售订单表（订单层面）
- **ConsigneeOrderProcessing**：配单流程记录表

### ConsigneeOrderProcessing 说明

- 关联某一个订单
- 记录谁配的这个单
- 开始和结束时间

**一个订单可能对应多条记录**：因为每次换配单员，旧的虽然会失效，但依然存在。

## 托管商功能逻辑

```
托管商户只提供货
    ↓
通过外部调货单 → 调货到公司仓库
    ↓
公司负责：销售 → 配单送货 → 售后
    ↓
每月按照单个商品生成月度佣金账单
    ↓
跟托管商户收取销售佣金
```

## 共享配单流程

```
给某公司创建合约
    ↓
合约开始前提：财务中对该公司进行押金收账
    ↓
货品入库 → 共享收货
    ↓
该公司的餐厅登录 → 进行共享下单
    ↓
下单完成 → 共享配单
```

## 米托管商订单体系（10 大功能）

1. 托管商户管理
2. 外部调货单
3. 订单查账
4. 订单收账
5. 月度佣金账单生成
6. 销售佣金结算
7. 配单送货
8. 售后处理
9. 库存扣减
10. 报表导出

**核心逻辑**：月度佣金对账 —— 每月 1 号定时任务汇总上月销售数据，生成佣金账单。

## 餐厅月度账单概念

| 字段 | 含义 |
|------|------|
| **debit** | 应收（借方）|
| **credit** | 实收（贷方）|
| **Balance** | 累计欠款 |

## CN / DN 账单

| 简写 | 全称 | 作用 |
|------|------|------|
| **CN** | Credit Note | 抵扣最后要付款的金额 |
| **DN** | Debit Note | 累加最后要付款的金额 |

## 合并账单（isOneBill）

查账接口返回 `bills` 数组字段，当订单存在售后问题时会产生多条数据。

当 `isOneBill` 为 `true` 时，导出 PDF 账单会合并在一个账单中。

可在订单详情（可创建售后）中进行切换。

## 分拣中心切换

客户下订单时根据截单时间决定分拣中心：

- **截单时间之前**：分拣中心是新加坡
- **截单时间之后**：分拣中心是马来西亚

## priceStructure 为什么是数组

因为某些商品有多个价格：
- 协议价
- 折扣价
- 原价

如果你买完了折扣价格的批次，还需要再买，这个时候只能购买原价的，就会存在多个价格。
