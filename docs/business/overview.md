# 业务整体概览

## 项目整体架构

项目分为多个业务模块，按客户端、员工端、管理后台、服务端划分：

```
客户端（APP/H5/小程序）→ 客户下单、查看订单
员工端（APP/H5）→ 配单、盘点、库存作业
管理后台（Web）→ 采购、审批、权限管理、报表
服务端 → 核心业务域、定时任务、实时推送
```

## 客户下单流程

客户端下订单时根据截单时间决定分拣中心：

- **截单时间之前**：分拣中心是新加坡
- **截单时间之后**：分拣中心是马来西亚

## 切菜业务

公司所有切菜都在**马来西亚加工车间**，针对新加坡和新山客户下的订单进行处理：

- **截单时间之前**：当天送到
- **截单时间之后**：第二天才能送

## 关系绩效

```
每月 1 号：结算上个月绩效，写入 PickerMerits 表
每天 7 点：更新配单绩效，员工端可查看
```

## 核心业务流转对应关系

### 采购流程

```
采购单收货入库 → 采购单列表点击更多进入 PO 流程
    ↓
上传 Invoice → 进入审核列表
    ↓
初审 → 复审
    ↓
财务管理中创建对应的发票 → 上传税务发票
```

### 共享配单流程

```
给某公司创建合约
    ↓
合约开始前提：财务中对该公司进行押金收账
    ↓
货品入库 → 共享收货
    ↓
该公司的餐厅登录 → 进行共享下单
    ↓
下单完成 → 共享配单
```

### 寄售订单流程

```
零售管理 → 寄售订单快捷下单
    ↓
导出模板 → 填入下单数量 → 上传模板 → 完成下单
    ↓
员工端 → 寄售商配单模块 → 配单
```

## 库存业务分类

### 大库 vs 周转库

**大库（仓库）出库**：

- 从仓库出货，减少仓库的库存
- 常规操作

**周转库（生产现场）出库**：

- 从生产现场的库存中出，减少生产现场的库存
- 因为有的产品没有放到仓库中，是直接进入生产现场库存的

**为什么不能全从大库出**？
因为有的产品没放到仓库中，相当于是从另一个地方直接进入到生产现场。

### 其他库存概念

- **退货入库**：减少生产的库存，增加仓库的库存（商品回到仓库）
- **产品入库**：给仓库人员用的
- **自助出库**：给生产现场的主管用的（晚上仓库没人上班时自己出库）

### 商品售卖库存是灵活的

例如土豆仓库中只有 10kg，但今天采购了 20kg 今天中午就能到，这时商品的售卖库存就可以显示为 30kg。

## 船业务

一个产品可能对应多个商品。产品出库时，需要将绑定到它身上的商品销售库存**都扣一遍**。

## 销售模式对比

| 概念       | 货的归属   | 谁销售     | 谁赚佣金 |
| ---------- | ---------- | ---------- | -------- |
| **寄售商** | EBUY 的货  | 寄售商     | 寄售商赚 |
| **托管商** | 托管商的货 | EBUY       | EBUY 赚  |
| **合作商** | 合作商的货 | 合作商自己 | 合作商   |

**合作商 vs 托管商的区别**：

- **合作商**：只是供货
- **托管商**：把整个业务都交给 EBUY 系统来管（销售、配单送货、售后）

## 托管商功能逻辑

```
托管商户只提供货
    ↓
通过外部调货单 → 调货到公司仓库
    ↓
公司负责：销售、配单送货、售后
    ↓
每月按照单个商品生成月度佣金账单 → 跟托管商收取销售佣金
```

## 餐厅月度账单（DebitCreditBalance）

- **debit（借方）**：应收
- **credit（贷方）**：实收
- **Balance**：累计欠款

## CN/DN 账单

| 简写   | 全称        | 作用                 |
| ------ | ----------- | -------------------- |
| **CN** | Credit Note | 抵扣最后要付款的金额 |
| **DN** | Debit Note  | 累加最后要付款的金额 |

## 合并账单（isOneBill）

查账接口返回 `bills` 数组字段，当订单存在售后问题时会产生多条数据。
当 `isOneBill` 为 `true` 时，导出 PDF 账单会合并在一个账单中。

在订单详情（可创建售后）中可以切换是否合并。

## 代理客户

指二手贩子，赚取中间商差价。

## 采购单完成状态

采购单只有在**上传凭证之后**才会变成已完成状态。

## 分拣中心切换规则

按客户下单时间：

- 截单时间之前 → 新加坡
- 截单时间之后 → 马来西亚

## 上传税务发票

- **TIN 码**：企业的唯一标识符
- **UUID**：税务系统中的 id

预览账单需要通过 `window.open()` 打开新页面，query 参数：

- `name`：当前账单 id
- `type`：表示在 PDF 中走哪个函数
