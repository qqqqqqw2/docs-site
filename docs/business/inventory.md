# 库存管理

## 库存分类

### 大库 vs 周转库（生产现场）

**大库（仓库）**：
- 从仓库出货，减少**仓库**的库存
- 常规入库存放位置

**周转库（生产现场）**：
- 从生产现场的库存中出，减少**生产现场**的库存
- 有的产品没放到仓库中，是直接进入生产现场库存的
- 所以不能全部都从大库出

## 生产库存相关表

| 表名 | 作用 |
|------|------|
| `SiteOrder` | 生产盘点单（按部门+配送日期）|
| `SiteOrderItem` | 盘点单中每个产品的记录（`quantity2B` = 实盘量）|
| `SiteRecord` | 生产库存变动记录 |
| `ProductionStockLoss` | 生产损耗记录 |
| `Good.stock` | 销售库存 |
| `GoodStockRecord` | 销售库存变动记录 |

## 关键字段

- **orgQuantity2B**：系统根据公式自动算出来的库存值（上次盘点量 + 入库 - 配单 - 退货 - 损耗），代表"**理论上应该有多少**"
- **quantity2B**：员工盘点时手动输入的实际数量，代表"**实际点了多少**"

## 生产库存计算公式

```
生产库存 =
    上次盘点量（orgQuantity2B）
  + 领货入库量（从 StockRecord 查，scope='sale'）
  - 配单消耗（从 OrderProcessingItem 查）
  - 寄售配单消耗（从 ConsigneeOrderProcessingItem 查）
  - 退货量（从 StockRecord 查，scope='return', returnType='site'）
  - 损耗量（从 ProductionStockLoss 查）
  - 船业务
```

## StockRecord 关键说明

领货出库、按产品出库、自助出库都会生成 `scope='sale'` 的 `StockRecord`。

## 自助出库和产品出库的整体逻辑

```
1. 查找/创建领货单
2. 创建 StockRecord
3. 扣仓位库存（Stock）
4. 扣实际库存（RealProduct）
5. 更新领货单状态
6. 条件：扣销售库存
7. 条件：扣生产库存
```

## 四种出库场景对比

### 1. 普通确认出库（不勾船用）

```
- 扣大库：StockRecord + Stock + RealProduct
- 不扣销售库存
- 不扣生产库存
```

### 2. 大库出（勾船用）

```
- 扣大库：StockRecord + Stock + RealProduct
- 扣销售库存：GoodStockRecord + Good.stock
- 扣生产库存：SiteRecord
```

### 3. 周转库出（勾船用）

```
- 不扣大库
- 扣销售库存：GoodStockRecord + Good.stock
- 扣生产库存：SiteRecord
```

### 4. 周转库出（不勾船用）

```
- 不扣大库
- 扣销售库存：GoodStockRecord + Good.stock
- 扣生产库存：SiteRecord
```

## 配单两种模式对比

### 批发配单模式

```
配单时只保存数据（pickType、giveQuantity）
   ↓
不扣库存
   ↓
可以反复修改，切换大库出/周转库出
   ↓
点"完成"时才一次性扣库存
```

- ✅ **优点**：编辑自由，不需要撤回
- ❌ **缺点**：完成前库存不准确

### 领货出库现有模式

```
点确认出库 → 立即扣库存
   ↓
编辑时 → 回退旧的再扣新的
```

- ✅ **优点**：库存实时准确
- ❌ **缺点**：编辑切换出库方式时撤回逻辑复杂

## 生产现场库存记录推导

**生产现场的库存记录** 其实就是 **产品库存记录（StockRecord）数据的累加**。

## 不会自动生成领货出库类型 SiteRecord 的情况

1. `isCountSiteOrder=false` → 不进 `saleTotal2B`，不生成对应 `SiteRecord`
2. 领货单被 bin 掉 → 跳过
3. 昨天根本没盘点单，直接建 `quantity2B=0` 空 item，所有今日变动都不会回溯生成 `SiteRecord`
4. 昨天有盘点单但没这个产品，也是建空 item 不回溯

## 退货入库

**业务含义**：减少生产的库存，增加仓库的库存（商品回到仓库）。

## 产品入库

**使用场景**：给仓库人员用的。

## 自助出库

**使用场景**：给生产现场的主管用的，有时候晚上仓库没人上班，就需要自己出库到生产现场。

## 产品入库到周转库存

**含义**：+ 生产库存

**注意**：操作指定时间没有入库，则会自动进入到生产库存。入库的时候可以选一个快速周转仓，这个就表示**周转入库**。

## 领货出库的周转库出

领货出库也可以选择**周转库出**，这个时候也需要**减生产库存**，因为这里的周转库出是直接给客户了，所以会减生产库存。

## 船业务库存扣减

船业务**一个产品可能对应多个商品**。产品出库时，需要将绑定到它身上的**商品销售库存都扣一遍**。

技术实现：用 MongoDB 事务保证原子性，任一商品扣减失败，整个事务回滚。

## 2s/2c 客户配单

像 2s、2c，针对这些客户进行配单时，**如果是从周转库出，则也需要扣减生产的库存**。

## 商品售卖库存的灵活性

商品的售卖库存是**灵活的**。

例如：土豆仓库中只有 10kg，但今天采购了 20kg 今天中午就能到，这个时候商品的售卖库存就可以显示为 **30kg**。
